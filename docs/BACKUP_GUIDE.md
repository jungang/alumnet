# 数据备份与恢复功能使用说明

## 功能概述

校友录系统现已支持完整的数据备份与恢复功能，可以备份和还原所有数据，包括：

- **数据库数据**：所有校友信息、留言、毕业照、杰出校友、班级名录等
- **上传文件**：照片、资料文档、班级照、老物件图片等所有上传的文件

## 访问备份功能

在管理后台登录后，通过左侧菜单访问"数据备份"功能页面。

URL：`http://your-domain/xyl/admin/backup`

## 功能说明

### 1. 创建备份

点击"创建备份"按钮，可以创建新的系统备份：

- **备份说明**：可以为备份添加描述信息（可选）
- **备份数据库**：勾选后导出PostgreSQL数据库所有数据
- **备份上传文件**：勾选后打包所有上传的文件（uploads目录）

备份文件将以 `backup_YYYYMMDD_HHMMSS.tar.gz` 格式保存在服务器 `backups` 目录中。

### 2. 备份文件列表

页面显示所有备份文件的列表，包括：

- 备份文件名
- 创建时间
- 文件大小
- 备份说明

### 3. 下载备份

点击"下载"按钮可以将备份文件下载到本地。下载的备份文件可以在其他环境中恢复使用。

### 4. 恢复备份

⚠️ **警告：恢复操作会覆盖当前数据库和文件！**

点击"恢复"按钮后：

1. 系统会弹出确认对话框
2. 确认后系统会：
   - 清空现有数据库（alumni_system schema）
   - 从备份文件恢复数据库
   - 备份现有uploads目录
   - 从备份文件恢复uploads目录
3. 恢复完成后系统会自动重新加载

### 5. 删除备份

点击"删除"按钮可以永久删除备份文件。删除前会弹出确认对话框。

⚠️ **警告：删除操作不可撤销！**

### 6. 上传备份

可以通过拖拽或点击上传的方式将备份文件上传到服务器：

- 支持格式：`.tar.gz`
- 最大文件大小：10GB
- 上传的备份文件会出现在备份列表中，可以用于恢复

## 备份文件结构

备份文件是一个 tar.gz 压缩包，包含：

```
backup_20250224_143000.tar.gz
├── database.sql          # PostgreSQL数据库导出
├── metadata.json         # 备份元数据
└── uploads/              # 上传文件目录
    ├── images/
    ├── documents/
    ├── photos/
    ├── vintage/
    └── temp/
```

## 技术实现

### 后端

- **服务文件**：`server/src/services/backupService.ts`
- **路由文件**：`server/src/routes/backup.ts`
- **API端点**：
  - `GET /api/backup/list` - 获取备份列表
  - `POST /api/backup/create` - 创建备份
  - `POST /api/backup/restore/:filename` - 恢复备份
  - `GET /api/backup/download/:filename` - 下载备份
  - `DELETE /api/backup/:filename` - 删除备份
  - `POST /api/backup/upload` - 上传备份

### 前端

- **页面组件**：`admin/src/views/BackupManage.vue`
- **路由配置**：`admin/src/router/index.ts`

### 依赖

- `archiver` - 创建tar.gz压缩包
- `tar` - 解压tar.gz文件
- `pg_dump` - PostgreSQL数据库导出（系统命令）
- `psql` - PostgreSQL数据库导入（系统命令）

## 最佳实践

### 备份策略

1. **定期备份**：建议每天或每周自动创建备份
2. **备份保留**：保留最近7-30天的备份
3. **异地备份**：定期下载备份文件到本地或其他存储

### 还原测试

定期在测试环境中验证备份文件的完整性和可恢复性。

### 存储管理

- 备份文件存储在服务器 `backups` 目录
- 定期清理过期的备份文件以释放存储空间
- 监控磁盘空间，避免备份导致磁盘满

## 系统要求

### PostgreSQL

- 服务器需要安装 PostgreSQL 客户端工具（`pg_dump` 和 `psql`）
- 备份用户需要有数据库的完全访问权限

### 磁盘空间

- 备份文件大小取决于数据库和上传文件的大小
- 需要足够的临时空间来创建备份（大约是备份大小的2倍）

## 注意事项

1. **备份期间**：系统正常运行，但大量数据备份可能会影响性能
2. **恢复期间**：系统会暂时不可用，建议在低峰期操作
3. **文件安全**：备份文件包含所有数据，请妥善保管
4. **权限控制**：只有管理员账号可以访问备份功能

## 故障排除

### 备份失败

- 检查 `pg_dump` 命令是否可用
- 检查数据库连接配置是否正确
- 检查磁盘空间是否充足

### 恢复失败

- 检查 `psql` 命令是否可用
- 检查备份文件是否完整（可以尝试解压验证）
- 查看服务器日志获取详细错误信息

### 上传失败

- 检查文件大小是否超过10GB限制
- 检查文件格式是否为.tar.gz
- 检查网络连接是否稳定
