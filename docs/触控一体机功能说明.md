# 吉林毓文中学校史馆智能校友查询系统深度功能说明书：基于AI/RAG技术与多模态交互的架构设计

## 1. 项目综述与设计愿景

### 1.1 项目背景与文化定位

吉林毓文中学作为一所承载着深厚历史底蕴与光荣革命传统的名校，其校史馆不仅是学校发展历程的陈列室，更是连接过去与未来、凝聚校友情感的精神家园。在数字化转型的时代浪潮下，传统的实体档案查询与静态图文展示已难以满足校友及访客日益增长的信息获取需求与情感交互体验。基于校方提出的建设需求，本项目旨在打造一套集"智能查询、情感连接、即时互动、服务延伸"于一体的《智能校友查询系统》。

该系统将部署于校史馆核心区域的大尺寸触控一体机上，作为校史馆的信息枢纽。设计核心不仅仅是数据的检索，更在于"激活"沉睡的档案。通过引入检索增强生成（RAG, Retrieval-Augmented Generation）技术与人工智能算法，系统将把枯燥的校友名录转化为生动的历史叙事，将静态的毕业照重构为可交互的时空记忆，从而实现^1^中所述的"兼具实用性、互动性和情感共鸣"的建设目标。

### 1.2 设计目标与核心价值

本说明书旨在深度细化系统功能设计，确保技术实现与用户需求的高度契合。

- **深度智能化（AI-Driven Intelligence）：** 突破传统关键词匹配的局限，利用RAG技术理解自然语言查询（如"查找80年代在文学领域有成就的校友"），提供精准且带有上下文的智能回复，并基于图谱算法实现校友关系的智能关联与推荐^1^。

- **沉浸式体验（Immersive Experience）：** 充分利用触控一体机的大屏优势，通过高清影像修复、3D老物件展示及流畅的动态UI设计，打造"时空长廊"般的视觉盛宴，增强校友的归属感。

- **隐私与安全（Privacy by Design）：** 严格遵循数据分级原则，针对公众访客与验证校友提供差异化的信息展示，通过动态脱敏技术保护校友个人隐私，构建安全可信的数字环境^1^。

- **交互人性化（Human-Centric Interaction）：** 考虑到跨年龄段用户的操作习惯，设计符合人体工学的触控手势、语音辅助输入及清晰的视觉导引，确保从耄耋老校友到在校新生均可无障碍使用。

### 2.2 体验旅程设计 (User Journey)

基于^1^的栏目设计，规划如下典型交互路径：

1.  **吸引阶段（Attract）：** 当无人操作时，一体机处于屏保模式，轮播经过AI修复的高清校园旧影与杰出校友寄语视频，通过视觉动态吸引访客驻足。

2.  **进入阶段（Engage）：** 访客点击屏幕，系统通过流畅的动效展开五大核心板块导航（核心查询、风采展示、时空长廊、互动寄语、校友服务）。

3.  **探索阶段（Explore）：**

    - *场景A（模糊查询）*：用户通过语音输入"帮我找一下1985年毕业的王老师的学生"，RAG引擎解析语义，展示相关结果。

    - *场景B（视觉浏览）*：用户在"时空长廊"通过手势滑动翻阅电子毕业照，点击人脸查看详情。

4.  **连接阶段（Connect）：** 验证校友身份后，查看同班同学通讯录，通过手机扫码获取联系方式（需授权）或加入校友会微信群。

5.  **离去阶段（Depart）：** 用户在留言板手写签名寄语，系统生成专属纪念海报供用户扫码带走，完成线上线下的闭环。

## 3. 核心功能模块深度设计

本章节将对^1^中提出的五大板块进行极尽详细的功能拆解与技术规范定义。

### 3.1 核心查询区：基于AI/RAG的智能检索引擎

这是系统的"大脑"，旨在解决传统查询系统"查不全、查不准"的痼疾。

#### 3.1.1 多模态输入层

为了适应触控大屏与不同年龄段用户，查询入口设计需具备极高的包容性。

- **智能手写输入：** 集成高精度手写识别SDK，支持连笔字、繁体字识别，专为习惯手写的老校友设计。

- **全双工语音交互：** 部署定向麦克风阵列，结合ASR（自动语音识别）技术。用户不仅可以说出姓名，还能进行多轮对话。

  - *示例*："我想找一位校友。" -\> 机器："请问他叫什么名字，或者是哪一届的？" -\> 用户："好像是90年代毕业的，后来去了清华大学。" -\> 机器："为您找到以下符合条件的校友\..."

- **虚拟键盘：** 大尺寸T9/全键盘布局，按键间距优化防止误触。

#### 3.1.2 检索增强生成 (RAG) 处理流程

传统数据库查询（SQL）只能处理结构化数据，无法理解"杰出"、"文艺界"等模糊概念。本系统引入RAG架构：

1.  **知识向量化：** 将校史馆内的非结构化文本（如校友传记、校刊文章、新闻报道）通过Embedding模型（如m3e-base）转化为高维向量存储于向量数据库（Vector DB）中。

2.  **语义检索：** 当用户输入"查找在医学界有贡献的校友"时，系统不再进行关键词匹配，而是计算输入语句与数据库中校友事迹的语义相似度。

3.  **生成式回答：** 检索到的相关片段被送入后台的大语言模型（LLM），模型生成一段流畅的综述："根据校史记录，吉林毓文中学在医学界的杰出校友包括\...其中张三校友曾主导\..."

4.  **结构化数据兜底：** 在生成文本的同时，下方同步列出精准匹配的校友卡片列表，确保信息的严肃性与准确性。

#### 3.1.3 高级筛选与组合搜索

针对专业用户（如寻找特定人才的校友会管理员），提供可视化的筛选器：

- **届别滑块：** 采用时间轴滑块设计，用户可拖动选择"1980-1990"年份区间。

- **标签云筛选：** 系统根据校友数据自动生成"行业标签"（政界、商界、学术、艺术），点击即可过滤。

- **院系/班级级联菜单：** 清晰的树状结构展示各年代的班级编制变化。

### 3.2 校友风采展示区：多媒体与榜样力量

该板块旨在通过高视觉冲击力的设计，弘扬毓文精神，展示校友成就^1^。

#### 3.2.1 杰出校友星空图 (Galaxy Visualization)

打破传统的列表式展示，采用"星空图"可视化隐喻。

- **交互逻辑：** 每一位杰出校友被渲染为一颗"星体"，星体的大小代表其热度或成就影响力。用户手指在屏幕上滑动，星空随之旋转。

- **分类轨道：** 星体分布在"政界"、"商界"、"学界"等不同轨道上。

- **详情展开：** 点击星体，镜头平滑推进，弹出该校友的数字档案卡。

  - *内容包括*：高清肖像、个人简介、成就时间轴、视频专访（支持4K流媒体播放）。

  - *推荐算法*：卡片底部自动推荐"与他同届的杰出人物"或"同一行业的校友"。

#### 3.2.2 动态新闻流

- **数据源聚合：** 后台爬虫定期抓取主流媒体关于知名校友的公开报道，或由管理员手动录入校友会动态。

- **瀑布流展示：** 图文混排，点击可阅读全文。对于重要奖项（如国家科技进步奖），界面会有专门的动效提示（如飘带、奖章动画）。

### 3.3 时空长廊：情感连接与数字复原

此区域是系统的"情感核心"，利用数字化手段复活沉睡的历史^1^。

#### 3.3.1 毕业照高清重构与人脸锚点

- **AI超分修复：** 对馆藏的黑白、泛黄的老毕业照进行数字化扫描，并利用GAN（生成对抗网络）进行去噪、上色、分辨率提升，使其在4K大屏上清晰可见。

- **交互式人脸识别：**

  - 系统预先对毕业照中的每一张人脸进行坐标标记（Face Tagging）。

  - 用户在触控屏上点击照片中的某个人物，系统会高亮该人物区域，并弹出浮层显示其姓名、现况（脱敏后）。

  - *寻人联动*：若该人物状态为"失联"，浮层中会显示"提供线索"按钮，引导知情校友补充信息。

#### 3.3.2 老物件数字博物馆 (3D Object Viewer)

- **3D摄影测量：** 对录取通知书、校徽、老课本、饭票等实物进行360度高精度扫描建模。

- **触控交互：** 用户可通过单指旋转、双指缩放来观察老物件的纹理细节（如纸张的褶皱、墨迹）。

- **多媒体叙事：** 每个物件旁配有"听故事"按钮，播放老校友或播音员录制的关于该物件的口述历史音频，营造沉浸式怀旧氛围。

#### 3.3.3 班级名录墙

- **电子书架：** 界面模拟图书馆书架，每一届是一个"卷宗"。

- **翻页动效：** 点击进入某班级，界面呈现逼真的纸质书翻页效果（Page Flip），增强历史厚重感。每页展示名单、班训、班主任寄语。

### 3.4 互动寄语区：UGC内容生产与实时互动

为了让校史馆"活"起来，必须引入用户的参与^1^。

#### 3.4.1 数字化留言板

- **原笔迹书写：** 利用触摸屏的高压感特性，还原毛笔或钢笔的书写笔锋，用户可选择不同的"墨水"颜色和"纸张"背景。

- **身份标签：** 留言发布时，系统自动关联用户的校友身份（如"1998届·张三"），增强留言的归属感。

- **弹幕模式：** 闲置状态下，精选的暖心留言会以半透明弹幕的形式在屏幕上方缓缓飘过。

#### 3.4.2 "寻找失联好友"征集令

- **发布机制：** 经过身份验证的校友可以发布寻人启事。

  - *必填项*：寻找对象姓名、班级、特征描述、回忆故事。

  - *隐私控制*：发布者的联系方式不直接公开，而是通过系统后台中转。

- **智能匹配推送：** 系统定期扫描数据库，若发现有新注册用户符合"寻人对象"的特征，会自动发送短信或微信通知："老同学XX在校史馆查询系统中给您留言了，快来看看吧。"

#### 3.4.3 校友视频祝福墙

- **自助录制亭模式：** 调用一体机摄像头，引导用户录制15-30秒的祝福短视频。

- **AI实时美颜与滤镜：** 提供"怀旧胶片"、"清新校园"等滤镜，以及自动美颜功能，降低用户上镜的心理负担。

- **内容审核：** 录制的视频不会立即发布，需经后台AI审核（过滤不雅画面）+人工复核后，才会上墙展示。

### 3.5 校友会服务区：线上线下服务闭环

将查询行为转化为实际的校友联络与回馈行动^1^。

#### 3.5.1 全球校友会地图

- **LBS可视化：** 基于GIS地图展示校友在全球的分布热力图。

- **组织导航：** 点击地图上的热点区域（如"上海"），弹出上海校友会的联系人、微信群二维码、定期活动地点。

#### 3.5.2 捐赠与回馈通道

- **项目展示：** 轮播学校的微捐赠项目（如"认领一棵树"、"资助一名学弟学妹"）。

- **扫码支付：** 提供聚合支付二维码，支持小额即时捐赠。捐赠成功后，大屏会即时播放感谢动画，并将捐赠者名字实时加入"爱心榜"。

#### 3.5.3 个人信息自主更新

- **移动端联动：** 屏幕上显示"我是校友，我要更新资料"的二维码。用户扫码进入手机端H5页面，进行身份认证后，可补充自己的最新工作单位、联系方式、上传近期生活照。这些数据经审核后将反哺回校史馆数据库。

### 4.2 数据脱敏与掩码规则 (Data Masking)

在前端展示层，系统根据当前登录的角色自动应用掩码规则，确保敏感数据"落地不裸奔"。

- **姓名：** 公开。

- **手机号：**

  - 访客视角：139 \*\*\*\* 8888

  - 校友视角（未授权）：139 \*\*\*\* 8888（点击申请查看）

  - 校友视角（已授权）：139 1234 8888

- **现居地：** 仅显示到城市级（如"北京市海淀区"），隐藏具体门牌号。

- **工作单位：** 默认显示，用户可设置为"保密"。

### 4.3 隐私握手协议 (Privacy Handshake)

当验证校友A想要查看校友B的联系方式，且校友B未完全公开时，系统执行以下"握手"流程：

1.  **发起请求：** 校友A点击"请求联系"，填写理由（如"老同学聚会"）。

2.  **中转通知：** 系统不直接提供B的电话，而是向B的微信/手机发送通知："您的同学\[张三\]在校史馆查询系统请求与您联系，理由是\..."

3.  **授权确认：** 校友B在手机端点击"同意"。

4.  **连接建立：** 系统向A发送B的联系方式，或直接交换微信名片。

### 5.2 关键技术选型

#### 5.2.1 前端交互层

由于需要极致的流畅度与复杂的动画效果（如3D老物件、星空图），前端建议摒弃传统的Web技术，采用 **Unity 3D** 或 **WPF (Windows Presentation Foundation)** 进行原生开发。

- **优势：** 支持GPU硬件加速，能够流畅运行4K分辨率下的粒子特效和3D模型渲染；对多点触控（Multi-touch）的支持更为底层和灵敏。

#### 5.2.2 后端AI核心 (RAG Stack)

- **大语言模型 (LLM)：** 考虑到校史数据的隐私性与响应速度，建议在本地边缘服务器部署私有化大模型（如阿里通义千问Qwen-72B或Llama-3的量化版本）。本地部署可确保数据不出校门，且不受公网波动影响。

- **向量数据库：** 选用 **Milvus** 或 **ChromaDB**。用于存储校友传记、校史文档的Embedding向量，支持毫秒级的亿级向量检索。

- **Embedding模型：** 使用 **BGE-M3** 等支持多语言、长文本的嵌入模型，确保语义理解的准确性。

#### 5.2.3 数据库设计

- **关系型数据库 (MySQL 8.0)：** 存储结构化的校友基础信息（姓名、学号、班级、联系方式）。

- **图数据库 (Neo4j)：** 构建"校友知识图谱"。存储"人物-班级"、"人物-事件"、"人物-人物（师生/同窗/夫妻）"的关系，用于实现智能推荐与关联挖掘。

## 6. 硬件环境与设备规范

优质的软件体验离不开高性能硬件的支撑。针对校史馆的特殊环境，提出以下硬件规范^1^。

### 6.1 主触控一体机规格

- **显示屏：**

  - 尺寸：86英寸或98英寸（根据墙面空间）。

  - 分辨率：真4K (3840 x 2160) @ 60Hz。

  - 亮度：≥450 nits（适应室内灯光，防眩光AG玻璃处理）。

  - 触控技术：红外或电容触控，支持20点触控，触摸精度±1mm，响应时间\<8ms。

- **OPS主机 (计算单元)：**

  - CPU：Intel Core i7-13700 或更高（高性能多核，保障AI推理预处理）。

  - GPU：NVIDIA GeForce RTX 4060Ti 16G（关键组件，用于Unity 3D渲染及本地LLM加速）。

  - 内存：64GB DDR5（大模型加载与多媒体缓存）。

  - 存储：2TB NVMe SSD（存储海量高清视频与模型文件）。

### 6.2 辅助外设

- **高清摄像头：** 4K广角镜头，支持HDR，用于视频留言与人脸识别。

- **麦克风阵列：** 4-6麦线性阵列，支持5米远场拾音、回声消除(AEC)、自动噪声抑制(ANS)。

- **身份识别模块：** 集成二代身份证阅读器与二维码扫描模组（支持微信/支付宝扫码）。

## 7. 数据治理与运营维护

系统上线后的数据鲜活度与安全性是长期运营的关键。

### 7.1 数据清洗与导入策略

校史数据往往来源杂乱（纸质档案、Excel散表、老系统数据）。

1.  **数字化扫描：** 使用高速扫描仪将纸质花名册电子化。

2.  **OCR识别与校对：** 利用OCR技术提取文字，并结合人工校对修正识别错误。

3.  **实体对齐 (Entity Resolution)：** 解决同名同姓问题。系统通过"届别+班级+姓名"作为唯一键值（Unique Key），若数据缺失则标记为"待人工核实"。

### 7.2 内容安全与审核机制

- **AI自动过滤：** 集成敏感词库与图像识别API，对用户提交的留言、视频进行实时扫描，拦截涉政、涉黄、暴力内容。

- **人工复审后台：** 管理员后台设有"待审核队列"，所有UGC内容建议采用"先发后审"或"先审后发"策略（视校方管控力度而定）。

### 7.3 系统监控与灾备

- **状态监控：** 部署Prometheus+Grafana，实时监控一体机的CPU温度、内存占用、网络延迟。若监测到程序崩溃，守护进程（Watchdog）将自动重启应用。

- **数据备份：** 实行"本地日备+云端周备"策略。数据库加密打包上传至私有云存储，防止勒索病毒或硬件损坏导致数据丢失。

## 8. 总结与展望

吉林毓文中学校史馆智能校友查询系统，不仅仅是一次技术层面的设备更新，更是一场关于"记忆与传承"的数字化革命。

本方案通过深度融合 **RAG人工智能技术**，让机器读懂了校史的字里行间；通过 **3D多媒体与触控交互**，让老照片后的青春岁月触手可及；通过严格的 **隐私分级设计**，在开放与保护之间找到了平衡点。

建成后的系统，将成为毓文中学校史馆中最具人气的"打卡点"。它是一本活着的校史书，随时等待着校友翻阅；它也是一座永不落幕的桥梁，让每一位走出校门的毓文人，无论身在何方，都能通过数据的脉络，感受到母校的温度与牵挂。这正是"智能校友查询系统"最核心的价值所在。
