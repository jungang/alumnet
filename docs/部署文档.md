## 首次部署

### 步骤1: 本地准备

```bash
# 确保已安装 pnpm
npm install -g pnpm

# 安装依赖
pnpm install

# Windows 用户使用 Git Bash 或 WSL 执行部署脚本
```

### 步骤2: 执行首次部署

**Windows (PowerShell):**
```powershell
# 构建所有项目
pnpm build:client
pnpm build:admin
pnpm build:server

# 创建服务器目录
ssh user@YOUR_SERVER_IP "mkdir -p /www/wwwroot/default/xyl/{client,admin,server/dist,server/uploads,logs}"

# 上传文件
scp -r client/dist/* user@YOUR_SERVER_IP:/www/wwwroot/default/xyl/client/
scp -r admin/dist/* user@YOUR_SERVER_IP:/www/wwwroot/default/xyl/admin/
scp -r server/dist/* user@YOUR_SERVER_IP:/www/wwwroot/default/xyl/server/dist/
scp server/package.json user@YOUR_SERVER_IP:/www/wwwroot/default/xyl/server/
scp server/.env.production user@YOUR_SERVER_IP:/www/wwwroot/default/xyl/server/
scp docker-compose.yml user@YOUR_SERVER_IP:/www/wwwroot/default/xyl/

# 启动Docker容器
ssh user@YOUR_SERVER_IP "cd /www/wwwroot/default/xyl && docker compose up -d"
```

**Linux/Mac:**
```bash
chmod +x deploy.sh
./deploy.sh init
```

### 步骤3: 配置宝塔Nginx

1. 登录宝塔面板 (http://YOUR_SERVER_IP:8888)
2. 进入 **网站** -> 选择站点 -> **设置** -> **配置文件**
3. 在 `server { }` 块内添加以下配置：

> ⚠️ **重要**: 必须使用 `^~` 前缀，避免被宝塔全局的 `.js/.css` 正则规则拦截

```nginx
# ============ 校友查询系统配置 START ============

# 校友查询系统 - 触控展示端
location ^~ /xyl {
    alias /www/wwwroot/default/xyl/client;
    index index.html;
    try_files $uri $uri/ /xyl/index.html;
}

# 校友查询系统 - 后台管理端
location ^~ /xyl/admin {
    alias /www/wwwroot/default/xyl/admin;
    index index.html;
    try_files $uri $uri/ /xyl/admin/index.html;
}

# 校友查询系统 - API反向代理
location ^~ /xyl/api {
    rewrite ^/xyl/api(.*)$ /api$1 break;
    proxy_pass http://127.0.0.1:3000;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    proxy_read_timeout 300s;
    client_max_body_size 50m;
}

# 校友查询系统 - 上传文件访问
location ^~ /xyl/uploads {
    alias /www/wwwroot/default/xyl/server/uploads;
    expires 30d;
    add_header Cache-Control "public, immutable";
}

# ============ 校友查询系统配置 END ============
```

4. 保存并重载Nginx

### 步骤4: 验证部署

```bash
# 检查API健康状态
curl http://YOUR_SERVER_IP/xyl/api/health

# 检查Docker容器状态
ssh user@YOUR_SERVER_IP "docker ps | grep xyl"
```

- 后台管理: http://YOUR_SERVER_IP/xyl/admin

## 故障排查

### 静态资源404

**症状**: 页面空白，控制台显示 `.js` 或 `.css` 文件 404

**原因**: 宝塔默认的 Nginx 配置有全局的 `location ~ .*\.(js|css)?$` 规则，优先级高于普通 location

**解决**: 确保使用 `location ^~` 前缀，而不是普通的 `location`

```nginx
# ❌ 错误
location /xyl { ... }

# ✅ 正确
location ^~ /xyl { ... }
```

### API 502错误

```bash
# 检查Docker容器是否运行
ssh user@YOUR_SERVER_IP "docker ps"

# 查看容器日志
ssh user@YOUR_SERVER_IP "docker logs xyl-server --tail=50"

# 重启容器
ssh user@YOUR_SERVER_IP "cd /www/wwwroot/default/xyl && docker compose restart"
```

### 前端路由404

**症状**: 直接访问子路由（如 `/xyl/search`）返回404

**原因**: Vue Router 的 history 模式需要服务器配置 fallback

**解决**: 确保 Nginx 配置中有 `try_files $uri $uri/ /xyl/index.html;`

### 数据库连接失败

```bash
# 检查环境变量
ssh user@YOUR_SERVER_IP "cat /www/wwwroot/default/xyl/server/.env.production"

# 测试数据库连接

```

## 技术要点

### 为什么使用 Docker？

CentOS 7 的 glibc 版本过低，无法直接运行 Node.js 18+。使用 Docker 可以：
- 隔离运行环境
- 使用最新的 Node.js 版本
- 简化部署流程

### 为什么使用 network_mode: host？

Docker 容器需要连接宿主机上的 PostgreSQL 数据库。使用 host 网络模式可以：
- 直接访问 localhost:5432
- 避免复杂的网络配置

### 热更新机制

- **前端**: 直接替换静态文件，刷新浏览器即可
- **后端**: 代码挂载为 volume，重启容器即可（无需重建镜像）
